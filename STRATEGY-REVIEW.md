# اتحاد الملاك — مراجعة تقنية واستراتيجية

> منصة إدارة العمارات السكنية — حالة المشروع الحالية، توصيات التطوير والتوسّع، وموقعه بين المنافسين في مصر والمنطقة والعالم.

## نظرة عامة

اتحاد الملاك هو نظام إدارة كامل لعمارة سكنية: الشقق وسكانها، الفواتير والتحصيل، طلبات الصيانة، الإعلانات، الرسائل، وصلاحيات ديناميكية بتتحكم في كل شاشة وكل زرار. المشروع بدأ كواجهة React من v0.app، وتحوّل بالكامل لـ Angular، وبُني له باك‑إند ASP.NET Core منفصل تمامًا من الصفر بمعمارية Clean Architecture — نفس أسلوب مشروع Afzaz ERP المرجعي، مع تحسينات حقيقية عليه (خصوصًا في الصلاحيات، اللي بقت بتتحقق من السيرفر مش من هيدر بيبعته الفرونت).

آخر إضافتين: (١) إصلاح جذري لمنظومة الصلاحيات عشان تتحكم فعليًا في الشاشات والقوائم في الفرونت، مش بس في الـ API، و(٢) دعم إدارة أكتر من عمارة من نفس حساب الأدمن.

**الحالة بصراحة**: النظام شغال ومُتحقق منه فعليًا على بيئة تطوير محلية. لسه محتاج بوابة دفع حقيقية، استضافة إنتاج، اختبارات آلية، ومونيتورينج.

## المعمارية التقنية

```
الواجهة       Angular 20 (Standalone + Signals)
              RTL كامل عربي/إنجليزي (ngx-translate، ٥٤٠ مفتاح)، لوحة إدارة + تطبيق مالك، Chart.js
                              ↓
API            ASP.NET Core 9 — JWT + Refresh Token، صلاحيات ديناميكية [HasPermission]، Serilog
Application    خدمات مستقلة عن EF Core عبر IApplicationDbContext
Domain         User · Building · Apartment · Invoice · MaintenanceRequest · RolePermission ...
                              ↓
قاعدة البيانات SQL Server — EF Core Migrations + DbUp، كل جدول مفلتر بـ BuildingId
```

**نظام الصلاحيات الديناميكي**: مصفوفة Screen × Action × Role كاملة قابلة للتعديل من واجهة الإدارة، بتتحقق من جدول `RolePermissions` عبر `[HasPermission("Screen.Action")]` مع كاش ١٠ دقايق. الفرونت بقى يفلتر الشاشات والقوائم بنفس الصلاحيات (بعد إصلاح 2026-07-11).

**تعدد العمارات**: حساب الأدمن الواحد يقدر يدير أكتر من عمارة. العمارة "النشطة" جزء من الجلسة (claim جوه التوكن) مش عمود ثابت على المستخدم.

## توصيات التطوير والتوسّع

### البنية التحتية
- **الاستضافة**: Azure App Service + Azure SQL (طبيعي لمشروع .NET)، أو VPS + Docker Compose في البداية لتقليل التكلفة.
- **CI/CD**: GitHub Actions لبناء واختبار كل Pull Request، ونشر تلقائي لبيئة staging.
- **اختبارات حقيقية**: xUnit للباك‑إند + Playwright رسمي مربوط بالـ CI بدل سكريبتات scratchpad المؤقتة.
- **الأمان**: نقل الأسرار لـ secrets manager، Rate Limiting على /auth/login و /auth/refresh.
- **المراقبة**: ربط Serilog بخدمة مركزية (Seq/Application Insights)، health-check endpoint.
- **التوسع الأفقي**: Redis بدل IMemoryCache لو أكتر من instance مستقبلاً.
- **نسخ احتياطي**: استراتيجية backup تلقائية قبل بيانات دفع حقيقية.

### المنتج
- **بوابة دفع فوري/Paymob حقيقية** — فوري عنده شراكات عقارية فعلية (أپ تاون، وادي دجلة).
- **إشعارات واتساب** — ولا منافس مصري وُجد بيستخدمها رسميًا، وهي القناة الأولى في مصر.
- **تصويت الجمعية العمومية** — فئة كاملة من المنتجات العالمية غير موجودة محليًا.
- **تقارير مالية جاهزة للتدقيق** — تسبق أي تنظيم مستقبلي محتمل.
- **رؤية موحّدة عبر العمارات (Portfolio View)** — تجميع بيانات كل العمارات في لوحة واحدة، مش بس تبديل.
- **إدارة البوابة والزوار** — للكمبوندات المسوّرة تحديدًا.

## المنافسون ومنتجات مشابهة

> بحث فعلي وقت الإعداد — بعض المصادر المصرية رجّعت محتوى غير مكتمل، فالثقة في MulakHub و7esba متوسطة‑لعالية، وفي Osool وitihadmulak.com أقل.

**مصر والمنطقة**: MulakHub (ملاك هَب) — الأقرب كمنافس مباشر (تسجيل مالي، صيانة، تقارير، اجتماعات). حسبة (7esba) — محاسبة فقط لاتحادات الشاغلين. أصول (Osool) — تفاصيل غير مؤكدة بالكامل. TAB-ERP — موديول ضمن ERP أكبر.

**عالميًا**: Buildium، AppFolio، PayHOA، Vantaca، Yardi Breeze Premier، TenantCloud — راجع الجدول الكامل في المستند التفاعلي. فئة كاملة من منتجات التصويت الإلكتروني (GetQuorum، ElectionBuddy) موجودة عالميًا وغائبة محليًا.

**ملهمة**: ADDA (الإمارات) — أقوى تشابه معماري، لوحة محفظة حقيقية لعمارات متعددة، مربوطة بمنصة "مؤلك" الحكومية. MyGatePass، NoBrokerHood (الهند)، RAY (السعودية/الإمارات).

## خارطة طريق مقترحة

1. **التفعيل التجاري** — بوابة دفع حقيقية + إشعارات واتساب
2. **الثقة والحوكمة** — تصويت الجمعية العمومية + تقارير مالية معتمدة
3. **الجاهزية للنمو** — استضافة إنتاج + CI/CD + تطبيق موبايل
4. **عمق العمليات** — لوحة محفظة موحّدة + إدارة بوابة وزوار
